# 🛸 Mission Control Workflow

이 워크플로우는 **하나의 복잡한 요청**을 여러 서브에이전트에게 분할하여 병렬 처리합니다.

## 사용 시나리오

### 시나리오 1: 신규 API 개발 (기획→배포)
```
사용자: "새로운 포트폴리오 분석 API 만들어줘"
```

**Step 1: 연구팀 소환** (browser_subagent)
```
Task: "포트폴리오 분석에 필요한 외부 API 조사
  - 주가 데이터 소스
  - 리스크 계산 공식
  - 벤치마크 지표"
```

**Step 2: 개발팀 소환** (코드 작성)
```
1. src/portfolio-api.step.ts 생성
2. src/analyze_portfolio_step.py 생성
3. Motia Flow 연결
```

**Step 3: 품질팀 소환** (테스트)
```
1. curl 테스트 실행
2. 코드 리뷰 체크리스트 확인
3. 품질 리포트 생성
```

**Step 4: 인프라팀 소환** (배포)
```
1. requirements.txt 업데이트 확인
2. docker-compose up 테스트
3. Hugging Face 배포 준비
```

---

### 시나리오 2: 버그 수정 + 문서 업데이트 (병렬)
```
사용자: "whale API 버그 수정하면서 README도 업데이트해줘"
```

**병렬 처리:**
```
┌─ 개발팀 → whale-api.step.ts 수정
└─ 연구팀 → README.md 업데이트
```

---

### 시나리오 3: 성능 최적화
```
사용자: "시총 유추 API가 너무 느려"
```

**Step 1: 품질팀** (성능 측정)
```
1. 현재 응답 시간 측정
2. 병목 지점 파악
```

**Step 2: 개발팀** (최적화)
```
1. 모델 캐싱 적용 확인
2. 불필요한 연산 제거
3. 비동기 처리 개선
```

**Step 3: 품질팀** (재측정)
```
1. 개선 후 응답 시간 비교
2. 성능 리포트 생성
```

## 에이전트 소환 트리거 키워드

| 키워드 | 소환 에이전트 |
|--------|--------------|
| 조사, 알아봐, 찾아봐 | 🔬 연구팀 |
| 만들어, 수정해, 추가해 | 💻 개발팀 |
| 테스트, 확인해, 점검해 | 🧪 품질팀 |
| 배포, 업로드, 푸시 | 🚀 인프라팀 |

## 병렬 작업 규칙

### Git Worktree 활용 (충돌 방지)
```powershell
# 병렬 작업 시 각 에이전트에게 별도 브랜치 할당
git worktree add ../bitcoin-ai-feature-1 feature/new-api
git worktree add ../bitcoin-ai-feature-2 fix/whale-bug
```

### 작업 완료 알림
```powershell
# 각 에이전트 작업 완료 시 OS 알림
[System.Windows.Forms.MessageBox]::Show("🧪 품질팀 테스트 완료!")
```
